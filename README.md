# RuStore MVP — Витрина мобильных приложений

Учебный проект, реализующий MVP-версию магазина приложений **RuStore** на современном стеке Android-разработки. Приложение отличается продуманным UX, использованием вкладок для навигации и устойчивой работой с сетью.

## ✅ Реализованный функционал

### Обязательные требования
*   **Умный онбординг:** Приветственный экран показывается только при первом запуске (состояние сохраняется в `SharedPreferences`).
*   **Главная витрина:** 
    *   Список приложений с иконками, названиями, рейтингом и категориями.
    *   **Fallback-механизм:** Если иконка не загрузилась, отображается стильная заглушка с первой буквой названия.
*   **Карточка приложения:** 
    *   Детальная информация: разработчик, возрастной рейтинг, полное описание.
    *   Горизонтальный список скриншотов с переходом в полноэкранный режим.
    *   Кнопка «Установить» (имитация загрузки).

### Опциональные требования и улучшения UI/UX
*   **Вкладки категорий (Tabs):** Быстрая фильтрация приложений прямо на главном экране (Все, Финансы, Игры и т.д.).
*   **Экран «Все категории»:** Отдельный список с количеством приложений в каждой категории. Доступен по кнопке в шапке.
*   **Полноэкранные скриншоты:** Поддержка свайпов и индикатора текущего изображения.
*   **Современный дизайн (Material 3):** 
    *   Использование теней, скруглений и градиентов.
    *   Адаптивная верстка под разные размеры экранов.

## 🛠 Технологический стек

*   **Язык:** Kotlin
*   **UI:** Jetpack Compose (Material 3)
*   **Архитектура:** MVVM (Model-View-ViewModel)
*   **Навигация:** Jetpack Navigation Component
*   **Загрузка изображений:** Coil (с обработкой ошибок)
*   **Хранение данных:** SharedPreferences, In-memory Repository

## 🚀 Как запустить

1.  Клонируйте репозиторий:
    ```bash
    git clone <ссылка_на_ваш_репозиторий>
    ```
2.  Откройте проект в **Android Studio** (Hedgehog или новее).
3.  Дождитесь синхронизации Gradle (требуется версия BOM `2023.10.01` или новее).
4.  Запустите приложение на эмуляторе (API 26+) или реальном устройстве.

### 💡 Как снова увидеть онбординг?
Чтобы протестировать первый запуск:
1.  Зайдите в настройки Android → Приложения → RuStoreMVP.
2.  Нажмите **«Очистить хранилище»**.
3.  Перезапустите приложение.

## 📂 Структура проекта

```text
com.rustore.mvp
├── data
│   ├── models       # Модели данных (AppItem, Category)
│   └── repository   # Источник данных (AppRepository)

│   ├── screens      # Экраны (Showcase, Detail, Categories, Onboarding)

└── MainActivity.kt  # Точка входа
```
